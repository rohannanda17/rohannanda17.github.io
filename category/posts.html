This post will be about using Unsupervised Techniques to find Semantically Similar Legal Provisions. In European Law, the Directives are implemented (or enforced) by national implementing measures (NIMs) as they are not directly applicable. These NIMs implement the directives in the national legal structure of each European Member State. Therfore, the language and terminologies of NIMs can be both similar and different from the directive. If a legal professional needs to find a particular provision of NIM which implements a particular article of directive, then he/she needs to read and interpret each provision. In this post, we will automate this this task by using text similarity techniques. Such systems act as a support tool to check if NIMs are legally compliant with directives or not.

Both Directives and NIMs are in a proprieatry XML format as shown above. The Figure above shows a particular European Directive being implemented by two national implementing measures (NIMs). We will develop a semantic similarity system to retrieve the relevant NIM provisions for a particular article of the directive. We will use 3 different approaches to compute semantic similarity :

  1. Vector Space Model based on Bag-of-Words Model
  2. Latent Semantic Analysis
  3. Knowledge-based approach (using a legal EuroVoc thesaurus)

def penn_to_wordnet(penn_tag):  #this function converts Penn Treebank tag to WordNet tag (so we can use WordNetLemmatizer)
    morphy_tag = {'NN':'n', 'JJ':'a','VB':'v', 'RB':'r'}
    try:
        return morphy_tag[penn_tag[:2]]
    except:
        return 'n'
    
def stopwords():
    from nltk.corpus import stopwords
    stopwds = stopwords.words('english') #NLTK's default stopword list
    stopwds_set = set(stopwds)  #convert stopwords list to set (so that we can do union operations with punctuation)
    punc_extra = set(['’','‘']) #adding extra punctuation characters missing in the default punctuation
    stopwords_plus_punct = stopwds_set.union(set(punctuation)) #merge stopwords with punctuation 
    stopwords_total = stopwords_plus_punct.union(punc_extra) # adding extra punctuation to the above set
    return stopwords_total


stopwords_total = stopwords()


def sentence_tokenizer(text):
    return nltk.sent_tokenize(text)

def word_tokenizer(text):
    return nltk.word_tokenize(text.lower())

def Lemmatizer(word,tag):
    return WordNetLemmatizer().lemmatize(word, penn_to_wordnet(tag))

def pre_processing(corpus):   #corpus is a list of documents or provisions
    text_corpus =[]
    for i in range(len(corpus)):
        text_corpus.append(corpus[i])
        text_corpus[i] = [sentence for sentence in sentence_tokenizer(text_corpus[i])] #sentence tokenizer
        text_corpus[i] = [word_tokenizer(sentence) for sentence in text_corpus[i]] #word tokenizer        
        text_corpus[i] = [item for sublist in text_corpus[i] for item in sublist] #flatten the list text_corpus
        text_corpus[i] = [item for item in text_corpus[i] if item.isalpha()] #taking strings with only alphabetic characters
        text_corpus[i] = [word for word in text_corpus[i] if word not in stopwords_total] # removing stopwords and punctuation
        text_corpus[i] = pos_tag(text_corpus[i]) #Part-of-speech tagging
        text_corpus[i] = [Lemmatizer(word,tag) for word,tag in text_corpus[i]] #Lemmatization
         
    return text_corpus

We also want to utilize the EuroVoc thesaurus and see if it can provide semantic knowledge necessary to capture similarity between directive articles and NIM provisions. EuroVoc is freely available to download in different formats. We use all the 127 microthesauri of EuroVoc to cover all possible domains for our corpus. We made use of the equivalence relationship between preferred and non-preferred terms in EuroVoc. This relationship defines the indexing term when more than one terms represent the same concept. The list of tokens obtained after pre-processing are compared with each element of the microthesauri. In case of a match the token in the corpus is replaced by the preferred indexing term as per the equivalence relationship of EuroVoc.

Next we implement Latent Semantic Analysis (LSA or LSI) on top of TF-IDF transform. LSI is able to club words which occur in similar contexts together by applying a SVD transform to represent relationships between terms in a reduced dimensional space. This is done to reduce the variability in word usage and uncover important latent relationships between terms which have same meaning but do not occuer in the same provision.
