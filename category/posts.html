This post will be about using Unsupervised Techniques to find Semantically Similar Legal Provisions. In European Law, the Directives are implemented (or enforced) by national implementing measures (NIMs) as they are not directly applicable. These NIMs implement the directives in the national legal structure of each European Member State. Therfore, the language and terminologies of NIMs can be both similar and different from the directive. If a legal professional needs to find a particular provision of NIM which implements a particular article of directive, then he/she needs to read and interpret each provision. In this post, we will address this task by using text similarity techniques.

Both Directives and NIMs are in a proprieatry XML format as shown above. The Figure above shows a particular European Directive being implemented by two national implementing measures (NIMs). We will develop a semantic similarity system to retrieve the relevant NIM provisions for a particular article of the directive.

from string import punctuation
import re
import string

stopwds = stopwords.words('english')
stopwds_set = set(stopwds)
#print(stopwds)

def penn_to_wordnet(penn_tag):     #this function converts Penn Treebank tag to WordNet tag
    morphy_tag = {'NN':'n', 'JJ':'a','VB':'v', 'RB':'r'}
    try:
        return morphy_tag[penn_tag[:2]]
    except:
        return 'n'
    

punc_extra = set(['’','‘'])

stopwords_plus_punct = stopwds_set.union(set(punctuation))
stopwords_total = stopwords_plus_punct.union(punc_extra)

def pre_processing(corpus):   #corpus is a list of documents or provisions
    text_corpus =[]
    for i in range(len(corpus)):
        text_corpus.append(corpus[i])
        text_corpus[i] = [sentence for sentence in nltk.sent_tokenize(text_corpus[i])]
        text_corpus[i] = [nltk.word_tokenize(sentence.lower()) for sentence in text_corpus[i]]
        text_corpus[i] = [item for sublist in text_corpus[i] for item in sublist] #flatten the list text_corpus
        text_corpus[i] = [item for item in text_corpus[i] if item.isalpha()]
        text_corpus[i] = [word for word in text_corpus[i] if word not in stopwords_total]
        text_corpus[i] = pos_tag(text_corpus[i])
        text_corpus[i] = [WordNetLemmatizer().lemmatize(word, penn_to_wordnet(tag)) for word,tag in text_corpus[i]]
    return text_corpus

Now, we have all the tokens in a Bag-of-Words format. Next we need to use Vector Space Model to transform the corpus so that we can compute text similarity. This is done by devloping a dictionary of the tokens in the corpus and then using a measure like Term Frequency or TF-IDF (Term Frequency-Inverse Document Frequency). TF-IDF has shown better results in semantic similarity and information retrieval tasks, so we will be utilizing that.

Now we have the similarity values in similarity_matrix. We utilized cosine similarity to compute similarity values over the TF-IDF corpus transform. Next we implement Latent Semantic Analysis (LSA or LSI) on top of TF-IDF transform. LSI is able to club words which occur in similar contexts together by applying a SVD transform to represent relationships between terms in a reduced dimensional space. This is done to reduce the variability in word usage and uncover important latent relationships between terms which have same meaning but do not occuer in the same provision.

def most_similar_nim_provision(art_index,matrix):

    most_similar_value,index = max([(v,i) for i,v in enumerate(reqd_list)]) # v is value and i is index
