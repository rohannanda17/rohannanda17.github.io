Supervised Short Text Similarity using Apache Spark

Published: Tue 24 April 2018
By Rohan

In posts.

In this post we will again implement Machine Learning classifiers (See previous post) to identify semantically similar legal provisions (Directive articles and NIM provisions). But this time we will do it using Apache Spark. Spark is useful for building Big Data applications. We will utilize PySpark which is the Spark Python API. Spark can also be programmed using Scala and Java. The objective is exactly same as the last post, ie. to classify whether two legal provisions are similar or not.

def read_csv(filename,encoding):
    with codecs.open(filename, "r",encoding) as csv_file:
        csv_reader = csv.reader(csv_file, delimiter='\t')
        true_all_columns = list(csv_reader)
        true_all_columns.pop(0)
    df = pd.DataFrame(true_all_columns) #Storing all columns list in a dataFrame for processing
    return df

The next step involves text pre-processing and transformation using TF-IDF features. We will not go in detail in pre-processing as it was well-discussed in the previosu supervised similarity post. We will utilze the Spark Pipelines. Both articles and provisions are processed separately as shown below. It is important to note that the class label(True/False) variable in Spark has to be numeric. Therefore, we will utilize a StringIndexer function to convert into a label column with 0 and 1 replacing False and True respectively. Also it is possible to have only one "features" column which can be used to predict the class label. Since we have to utilize two different texts (articles and provisions) we utilize VectorAssembler which merges the individual features of both articles and provisions into the feature column.

from pyspark.ml import Pipeline
from pyspark.ml.feature import HashingTF, IDF
from pyspark.ml.classification import LogisticRegression, NaiveBayes, LinearSVC
from pyspark.ml.feature import Tokenizer, StopWordsRemover, CountVectorizer,RegexTokenizer
from pyspark.ml.feature import StringIndexer, VectorAssembler
from pyspark.sql.types import IntegerType

tokens_arts = Tokenizer(inputCol="article", outputCol="tokens_arts") #Tokenizing directive articles
tokens_provs = Tokenizer(inputCol="provision", outputCol="tokens_provs") #Tokenizing NIM provisionss

#Computing Term-Frequencies
hashingTF_arts = HashingTF(inputCol="tokens_arts", outputCol="rawFeatures_arts", numFeatures=1000)
hashingTF_provs = HashingTF(inputCol="tokens_provs", outputCol="rawFeatures_provs", numFeatures=1000)

#Computing Inverse Document Frequencies
idf_arts = IDF(inputCol="rawFeatures_arts", outputCol="features_arts", minDocFreq=5)
idf_provs = IDF(inputCol="rawFeatures_provs", outputCol="features_provs", minDocFreq=5)

#Converting categoric labels to numeric
label_stringIdx = StringIndexer(inputCol = "Category", outputCol = "label")


vecAssembler = VectorAssembler(inputCols=["features_arts", "features_provs"], outputCol="features")

label_stringIdx = StringIndexer(inputCol = "Category", outputCol = "label")


p1 = Pipeline(stages=[tokens_arts, hashingTF_arts, idf_arts]) #Pipeline p1 for Articles
p2 = Pipeline(stages=[tokens_provs, hashingTF_provs, idf_provs]) #Pipeline p2 for Provisions

p1_fit = p1.fit(data)
d1 = p1_fit.transform(data)

p2_fit = p2.fit(d1)
d1 = p2_fit.transform(d1)


p3 = Pipeline(stages=[label_stringIdx,vecAssembler]) #Pipeline p3 for numeric label and combining features

p3_fit = p3.fit(d1)


dataset = p3_fit.transform(d1) #dataset containing combined features from both articles and provisions

from pyspark.ml.evaluation import BinaryClassificationEvaluator

#Logistic Regression
lr = LogisticRegression(maxIter=20, regParam=0.3, elasticNetParam=0)
lrModel = lr.fit(trainingData)
predictions = lrModel.transform(testData)
evaluator = BinaryClassificationEvaluator(rawPredictionCol="prediction")
print("Logistic Regression Accuracy:%f" %evaluator.evaluate(predictions))
